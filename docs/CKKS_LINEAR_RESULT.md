# CKKS Linear logreg_small Result

---

# English

This document summarizes the current encrypted CKKS evaluation result for the linear part of `logreg_small`.

The result is generated by:

```bash
./scripts/run_ckks_linear.sh
```

## 1. Experiment

The current CKKS linear experiment evaluates the following expression over encrypted inputs:

```text
z = 0.8*x1 - 0.5*x2 + 1.2*x3 - 0.3
```

This is the linear part of the `logreg_small` benchmark.

The full polynomial inference path is:

```text
z = 0.8*x1 - 0.5*x2 + 1.2*x3 - 0.3
y = 0.5 + 0.197*z - 0.004*z^3
T = 0.5
```

This step only evaluates `z`.

## 2. Current Scope

Implemented:

- Lattigo CKKS parameter initialization
- key generation
- input encryption
- ciphertext-constant multiplication
- ciphertext addition
- decryption
- decoding
- plaintext reference comparison
- CSV report export
- Markdown report export
- CLI experiment
- reproducibility script

Not yet implemented:

- encrypted `z^2`
- encrypted `z^3`
- encrypted polynomial output `y`
- CKKS decision comparison against threshold `T`
- explicit scale/rescale management
- rotation-based packed evaluation
- runtime and level-consumption comparison across scale policies

## 3. Result

Current command:

```bash
go run ./cmd/flipguard -experiment ckks_linear
```

Current representative output:

```text
sample=0 plain_z=0.6100000000 ckks_z=0.6100000772 abs_error=0.0000000772
sample=1 plain_z=-0.3000000000 ckks_z=-0.3000000379 abs_error=0.0000000379
sample=2 plain_z=-0.0500000000 ckks_z=-0.0500000064 abs_error=0.0000000064
sample=3 plain_z=0.0950000000 ckks_z=0.0950000121 abs_error=0.0000000121
```

Aggregate result:

```text
samples = 4
max_abs_error = 0.0000000772
mean_abs_error = 0.0000000334
initial_level = 6
final_level = 6
log_default_scale = 45
```

## 4. Interpretation

This result confirms that FlipGuard can now execute the linear part of the benchmark using actual CKKS ciphertext operations.

The current backend is still preliminary. In particular, scalar multiplication currently uses a temporary decode-scale correction:

```text
ckks_z = raw_ckks_z / 2^log_default_scale
```

This correction is acceptable as a backend probe, but it is not the final CKKS scale-management design.

The next implementation step should replace this temporary correction with explicit CKKS scale and rescale handling.

## 5. Research Meaning

This step is important because the project has moved from simulation-only evaluation to actual encrypted CKKS computation.

Current progression:

```text
plain simulation
  -> scalar CKKS roundtrip
  -> vector CKKS input roundtrip
  -> encrypted CKKS linear evaluation
```

The next research milestone is:

```text
encrypted linear z
  -> encrypted polynomial y
  -> encrypted decision stability measurement
  -> comparison with FlipGuard schedules
```

## 6. Limitation

The current result does not yet prove that FlipGuard improves CKKS runtime or level consumption.

It only proves that the current repository can perform encrypted CKKS evaluation of the linear component with small output error.

Further work is required to connect FlipGuard schedules to real CKKS scale policies.
